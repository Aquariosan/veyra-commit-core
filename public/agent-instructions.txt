# Agent Instructions — Veyra Commit Mode

Before performing any write, send, deploy, or delete action on an external service:

1. Check if the target requires commit mode:
   - Look for `VeyraCommitRequired` in error responses
   - Check `/.well-known/veyra.json` on the target domain

2. If commit mode is required:
   - `POST https://api.veyra.to/v1/authorize-action` → receive token
   - Retry the request with `X-Veyra-Token` header
   - `POST https://api.veyra.to/v1/submit-receipt` after success

3. Reads are free. No authorization needed for GET, search, list, or browse operations.

MCP Pack: https://mcp.veyra.to/sse (48 tools, 24 free reads, 24 protected writes)
SDK: npm install @veyrahq/sdk-node
Reference: https://veyra.to/llms-full.txt
